# 教育特化型RAG対話メディア　システム仕様書

**対象論文**：AIとの対話を通じた教育関係者の意見言語化支援 ― 教育特化型RAGと対話モードを備えた対話型メディアの設計と予備的評価 ―（AsiaEdu 2026）
**版**：v1.0（2026-06-20）
**本番URL**：https://edumatch-five.vercel.app
**リポジトリ**：https://github.com/studypark2011/edumatch

> 本書は、AsiaEdu 2026 予備調査で用いた「対話パート」アプリ（検証用に必要機能のみを実装した版）の仕様を記述する。意見投稿導線（論文3.4）を含むフル機能版は別の本番環境に実装されており、本検証版では測定対象外のため除外している。

## 1. システム概要

教育関係者（教師・学校管理職・教育委員会/自治体担当者・民間教育事業者）が、教育とテクノロジーに関する正解のない論点について AI と対話し、自らの意見を形成・言語化することを支援する Web ベースの対話型メディアである。

設計思想は、AI を「答えを与える存在」ではなく、利用者が自ら考えるための **思考の足場（scaffolding）** として位置づける点にある。AI は結論を提示せず、論点を分解し、判断の前提となる根拠（公的文書）を補い、多様な立場を整理して提示する役割に限定される。

本検証版は、Google フォーム（事前・事後アンケート）の途中で行う「AI との対話」を担う。フォームが自己効力感等を測定し、本アプリが対話体験（RAGあり/なし）を提供する。

## 2. 全体アーキテクチャ

```
[参加者ブラウザ]
     │  HTTPS
     ▼
[Next.js 16 (App Router) on Vercel]
  ├─ 参加者UI（テーマ選択 / 対話 / モード切替）
  ├─ 管理UI（文書・モード・実験設定・CSV出力）
  └─ API Routes（Node.js runtime）
        ├─ /api/session   セッション生成・条件解決
        ├─ /api/chat      対話（ストリーミング）＋RAG
        └─ /api/admin/*   管理機能
     │                         │
     │ service role            │ 埋め込み/生成
     ▼                         ▼
[Supabase Postgres + pgvector]   [Anthropic Claude / OpenAI Embeddings]
  themes / documents / document_chunks(vector)
  modes / experiment_config
  participants / conversations / messages
```

- フロントエンドとバックエンド（API Routes）は Next.js に同居し、Vercel にデプロイ。
- データ永続化・ベクトル検索は Supabase（マネージド PostgreSQL ＋ pgvector 拡張）。
- 対話生成は Anthropic Claude、文書/クエリのベクトル化は OpenAI の埋め込みモデル。

## 3. 技術構成

| 層 | 採用技術 | 備考 |
|---|---|---|
| フロントエンド | Next.js 16（App Router）/ React 19 / TypeScript / Tailwind CSS | サーバコンポーネント中心 |
| ホスティング | Vercel | サーバレス関数（Node.js runtime） |
| データベース | Supabase（PostgreSQL 15＋ `pgvector`） | RLS有効、サーバは service role で操作 |
| 対話 LLM | Anthropic **Claude Sonnet 4.6**（`claude-sonnet-4-6`） | ストリーミング応答、`max_tokens=1024` |
| 埋め込み | OpenAI **text-embedding-3-small**（出力 **1024 次元**） | Matryoshka により 1024 次元に縮約。Voyage AI への切替も実装 |
| ベクトル検索 | pgvector（HNSW インデックス、コサイン距離） | RPC `match_chunks` |

> 埋め込みのみ OpenAI を用いるのは、対話に用いる Anthropic が埋め込み API を提供しないため。次元はスキーマ（`vector(1024)`）と一致させ、プロバイダを切り替えても整合するよう統一している。

## 4. データモデル

主要テーブル（PostgreSQL）。識別子は UUID、時刻は `timestamptz`。

| テーブル | 役割 | 主なカラム |
|---|---|---|
| `themes` | 対話テーマ（お題） | `slug`, `title`, `intro`（お題提示文）, `rag_tag`（参照文書タグ） |
| `documents` | RAG知識基盤の文書 | `title`, `source_type`(url/file/text), `tags[]`, `status`, `chunk_count` |
| `document_chunks` | 文書チャンクと埋め込み | `document_id`, `chunk_index`, `content`, `tags[]`, `embedding vector(1024)` |
| `modes` | 対話モード定義 | `key`, `name`, `description`, `system_prompt`, `sort_order` |
| `experiment_config` | 群×テーマの実験条件 | `theme_slug`, `group_label`, `displayed_mode`, `dialogue_mode_key`, `rag_enabled` |
| `participants` | 参加者（匿名） | `group_label`, `role`, `ai_usage`（※PII を含まない） |
| `conversations` | 対話セッション | `participant_id`, `theme_slug`, `group_label`, `displayed_mode`, `dialogue_mode_key`, `rag_enabled` |
| `messages` | 各発言 | `conversation_id`, `role`(user/assistant), `content`, `dialogue_mode_key`, `citations`(jsonb) |

インデックス：`document_chunks.embedding` に HNSW（`vector_cosine_ops`）、`document_chunks.tags` に GIN、`messages.conversation_id` に B-tree。

## 5. 教育特化型RAG（論文 3.2）

### 5.1 取り込みパイプライン
1. **本文抽出**：URL（PDF は pdf-parse、HTML は本文抽出）／ファイル（抽出済みテキスト）／直接入力。
2. **チャンク分割**：段落境界を尊重しつつ **1チャンク約800文字・オーバーラップ120文字** に分割。
3. **ベクトル化**：OpenAI text-embedding-3-small（1024次元）で 96 件ずつバッチ埋め込み。
4. **保存**：`documents`（メタ）と `document_chunks`（本文＋ベクトル）に保存し、状態を `ready` に更新。

文書には **タグ**（`theme1` / `theme2` / 複数可）を付与し、テーマ単位で検索範囲を制御する。

### 5.2 検索（Retrieval）
- 利用者の発話を同一モデルでベクトル化し、PostgreSQL 関数 `match_chunks(query_embedding, filter_tag, match_count)` を呼び出す。
- テーマの `rag_tag` でタグ絞り込みのうえ、コサイン類似度（`1 - (embedding <=> query)`）上位 **6 件** を取得。
- 取得チャンクを「【参考資料】」ブロックとしてシステムプロンプト末尾に差し込み、引用元（文書名・URL）を明示するよう指示する。

### 5.3 RAGあり/なしの差（RQ2）
同一モデル・同一プロンプト・同一対話モードのまま、**`rag_enabled` フラグのみ**で参考資料ブロックの有無を切り替える。RAGあり時は応答に参照資料が併記され、根拠が明示される。

## 6. 対話モード（論文 3.3）

利用者の意見形成の段階に応じた 3 モードを提供。各モードの振る舞いは `modes.system_prompt` で規定し、管理画面から編集可能。並びは段階順（ナビゲーター→ディスカッション→ディベート）で、各役割は対話画面に常時表示される。

| モード | 対象段階 | 役割 |
|---|---|---|
| ナビゲーター | テーマがよく分からない | 背景・論点をかみ砕いて整理し、考える出発点を整える。RAGが主に機能 |
| ディスカッション | 考えを整理したい | 意見を要約・構造化し、合意形成的に深める |
| ディベート | 考えを試したい | あえて対立する立場から反論し、意見の根拠を問い直させる |

すべてのモードに共通の前置き（日本語・簡潔・足場かけ優先・結論を押し付けない）を毎回付与する。RAGあり時は各プロンプト末尾に参考資料ブロックを追加する。

> 検証では順序効果・テーマ効果の混入を避けるため、対話モードは全条件で `navigator` に固定（実験条件は管理画面で変更可能）。利用者によるモード切替機能（フル機能）も画面上は利用可能。

## 7. 対話処理フロー

1. **セッション開始**（`POST /api/session`）：群（X/Y）とテーマを受け取り、`experiment_config` から条件（表示モードA/B・対話モード・RAG有無）を解決。`participants` と `conversations` を作成。**RAG有無はクライアントに返さない（盲検）**。
2. **発話送信**（`POST /api/chat`）：会話IDと発話を受け取り、サーバ側で会話の条件を取得。RAGあり時は検索を実行。`BASE_PREAMBLE ＋ テーマ文脈 ＋ モードのsystem_prompt ＋（RAG参考資料）` をシステムプロンプトとして Claude にストリーミング要求。
3. **応答配信**：NDJSON で `meta`（引用一覧）→ `delta`（逐次本文）→ `done` を返却。
4. **ログ保存**：user/assistant の各発言を `messages` に保存（assistant には引用 `citations` を含む）。

## 8. A/B 実験制御（論文 表1・カウンターバランス）

被験者内A/Bデザイン。回答者には中立的呼称「モードA／モードB」のみを提示し、中身（RAGあり/なし）を群ごとに入れ替える。

| 群 | テーマ1（生成AIの年齢） | テーマ2（校則と生徒の意見反映） |
|---|---|---|
| 群X | モードA = RAGあり | モードB = RAGなし |
| 群Y | モードA = RAGなし | モードB = RAGあり |

- 集計時：**RAGあり** = 群Xテーマ1 ＋ 群Yテーマ2／**RAGなし** = 群Xテーマ2 ＋ 群Yテーマ1。
- 条件は `experiment_config` に保持し、管理画面から変更可能。`conversations` 行に条件が記録されるため、後から条件別に集計できる。

## 10. データ収集・出力

- 対話はすべて `conversations`（条件）と `messages`（発言）に記録。
- 管理ダッシュボードから CSV を出力可能。
  - **メッセージCSV**：`conversation_id, participant_id, group, theme_slug, displayed_mode, rag_enabled, msg_role, msg_dialogue_mode, content, has_citations, created_at`
  - 群・テーマ・RAG有無の列を含むため、RAGあり/なしの組み替え集計に対応。

## 11. プライバシー・研究倫理

- 氏名・メールアドレス等の個人を特定する情報は一切収集しない（完全匿名）。
- 参加者属性は群・立場・AI利用頻度のみ（フォーム側で取得、本アプリは群のみ受領）。
- 管理画面はパスワード認証（httpOnly cookie）で保護。サーバ秘密鍵（service role）・各APIキーは環境変数で管理し、公開しない。

---

## 12. 制約・限界

- PDF 本文抽出は pdf-parse に依存し、**フォントが特殊エンコード（ToUnicode 無し）または画像ベースの PDF はテキスト抽出ができない**（該当文書は OCR が別途必要）。
- 埋め込みに OpenAI を用いるため、日本語クエリと英語文書の**言語間検索は相対的に弱い**。本研究では知識基盤を日本語資料に統一した。
- サーバレス関数のタイムアウトのため、極端に大きな文書の取り込みは専用スクリプト（`scripts/ingest-pdf.mjs`）でローカルから実行する。

## 13. 論文記載用サマリー（そのまま引用可）

> 本システムは Next.js（App Router）で実装し Vercel 上にデプロイした Web アプリケーションである。データ永続化と検索には Supabase（PostgreSQL ＋ pgvector）を用いた。対話生成には Anthropic Claude（Sonnet 4.6）を、教育特化型 RAG の文書・クエリのベクトル化には OpenAI の埋め込みモデル（text-embedding-3-small、1024 次元）を用いた。教育分野の公的文書を約800文字単位でチャンク化して埋め込み、利用者の発話に対しコサイン類似度上位の文書片を検索して応答の根拠とした。対話モード（ナビゲーター／ディスカッション／ディベート）はシステムプロンプトにより規定した。評価では、同一モデル・同一プロンプト・同一モードのまま教育特化文書の参照有無のみを切り替え、被験者内でカウンターバランスして比較した。回答者には条件を中立的呼称（モードA／B）でのみ提示し、教育特化RAGの有無は伏せた。
